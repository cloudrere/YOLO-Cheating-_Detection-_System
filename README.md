# 作弊监控系统 · Cheating Monitor System

基于计算机视觉的考试/在线监考作弊行为识别系统。支持**图片**、**视频**、**摄像头实时**三种检测模式,识别头部姿态偏离、瞳孔/视线偏离、手机使用、离座、多人围坐等可疑行为,并以阈值驱动的警报机制生成证据截图与 Excel 报告。

---

## 一、功能特性

| 分类 | 特性 |
|---|---|
| 检测维度 | 头部姿态 (6 方向) · 瞳孔方向 (4 方向) · 手机使用 · 离座 · 多人 · 闭眼 |
| 检测来源 | 图片 · 视频文件 · 摄像头实时 |
| 控制 | 开始 / 暂停 / 继续 / 停止 / 清除 |
| 阈值 | 5 类违规各自独立阈值 (秒),可自由调节 |
| 性能 | 视频跳帧(默认 3x 加速),逐帧检测可选 |
| 校准 | 摄像头 30 帧显式校准 · 视频首帧静默自校准 · 图片两步自校准 |
| 输出 | 违规截图自动归档 + Excel 报表 (明细+统计) |
| UI | 检测前/检测后双画面 · 5+1 统计卡片 · 彩色日志 · 事件明细表 |
| 稳定性 | 检测耗时在独立 QThread · UI 不卡顿 · 会话切换自动重置状态 |

## 三、安装

### 1. Python 环境

需要 Python 3.7+。推荐虚拟环境:

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Linux / macOS
source venv/bin/activate
```

### 2. 安装依赖

```bash
pip install PyQt5 opencv-python numpy openpyxl dlib
```

**关于 dlib**:若 `pip install dlib` 编译失败,可改用预编译 wheel:

```bash
pip install dlib-bin
```

### 3. 下载模型文件

dlib 68 点人脸关键点模型,放到项目根目录或 `model/` 子目录:

- 下载: <http://dlib.net/files/shape_predictor_68_face_landmarks.dat.bz2>
- 解压后得到 `shape_predictor_68_face_landmarks.dat` (约 100 MB)

模块会自动在以下路径查找:

1. 当前工作目录
2. `./model/` 子目录
3. 与 `.py` 文件同目录
4. 与 `.py` 文件同目录下的 `model/` 子目录

## 四、快速开始

```bash
python cheating_monitor_system.py
```

### 典型流程

1. **选择来源** — 点左侧 `① 图片检测` / `② 视频检测` / `③ 摄像头检测`。选择后,"检测前"窗口立即显示原始画面(摄像头为持续预览)
2. **调整阈值**(可选)— 默认值适合大多数场景
3. **点击 `▶ 开始检测`** 进入检测
   - 摄像头:先做 30 帧头部校准(约 1 秒,请保持正脸)
   - 视频/图片:首帧静默自校准
4. **随时 `⏸ 暂停`** 或 `■ 停止`
5. **点击 `💾 保存结果`** 导出 Excel 报表

## 六、参数配置

### 警报阈值 (Alert Thresholds)

| 参数 | 默认 | 说明 |
|---|---|---|
| 头部阈值 | 3.0 秒 | 头部持续偏离屏幕超过此时长才触发头部违规 |
| 瞳孔阈值 | 3.0 秒 | 瞳孔/视线持续偏离超过此时长才触发瞳孔违规 |
| 手机阈值 | 3.0 秒 | 检测到手机持续超过此时长才触发手机违规 |
| 离座阈值 | 2.0 秒 | 画面中持续无人脸超过此时长触发离座违规 |
| 多人阈值 | 1.0 秒 | 画面中持续 ≥2 张人脸超过此时长触发多人违规 |

**阈值越小越敏感,越大越宽松**。推荐:

- 严格考试场景:全部调低到 1.0-2.0 秒
- 普通场景:使用默认值
- 宽松环境(允许短暂转头):头部/瞳孔调高到 5.0 秒

### 视频参数

| 参数 | 默认 | 说明 |
|---|---|---|
| 视频跳帧 | 2 帧 | 视频每检测 1 帧跳过 N 帧。0=逐帧, 2=3x 加速, 5=6x 加速 |

跳帧不影响摄像头和图片模式。

## 七、违规类型说明

### 五类违规 (Violation Categories)

| key | 中文 | 触发条件 | 严重度 |
|---|---|---|---|
| `head` | 头部违规 | 头部方向 ∈ {Left, Right, Up, Down, Tilted} | 中 |
| `eye` | 瞳孔违规 | 瞳孔方向 ∈ {Left, Right, Up, Down} | 中 |
| `mobile` | 手机违规 | 画面中检测到手机 | 高 |
| `absence` | 离座违规 | 画面中无人脸 | **高** |
| `crowd` | 多人违规 | 画面中 ≥ 2 张人脸 | **极高** |

### 非违规状态 (不计数)

| 状态 | 处理 |
|---|---|
| `Looking at Screen` | 正常,不计数 |
| `Eyes Closed` | 仅显示橙色提示,不计入瞳孔违规(避免眨眼误报) |

### 阈值的设计逻辑

- 前三类(头部/瞳孔/手机)采用**较长阈值** (3 秒):允许正常的短暂动作 (转头看桌面、揉眼睛、短暂拿手机)
- 后两类(离座/多人)采用**较短阈值** (1-2 秒):严重异常,不需要过多缓冲

## 九、技术细节

### 模块边界

- **`eye_movement.py`** — 瞳孔检测 (Otsu 自适应阈值 + 多边形遮罩) + EAR 闭眼检测 + 多数投票状态平滑
- **`head_pose.py`** — dlib 68 关键点 + solvePnP 位姿解算 + 动态相机矩阵 + hysteresis 死区
- **`mobile_detection.py`** — 用户自己的手机检测实现(通常基于 YOLO 等)
- **`cheating_monitor_system.py`** — PyQt5 GUI + 三线程架构 (Preview / Detection / UI)

### 线程架构

```
 UI 线程           PreviewThread         DetectionThread
   │                    │                      │
   ├─ 选择来源 ────────>│                      │
   │                    │── 读取帧 ───> 显示   │
   │                    │                      │
   ├─ 开始检测 ────stop >│                      │
   │                    X                  ───>│── 读取+检测+警报 ──> UI
   │                                           │
   ├─ 暂停/继续 ──────────────────────────────>│
   │                                           │
   └─ 停止 ───────────────────────────────────>X
```

### 校准逻辑

- **摄像头**: 前 30 帧显式校准,屏幕提示 "Calibrating..." 要求保持正脸
- **视频**: 静默取第一帧角度作为校准基准(假设视频第一帧为正脸)
- **图片**: 两步自校准 — 先取角度,再以该角度为基准判方向 (单张图片上头部方向判断参考意义有限)

### 会话隔离

每次"开始检测"会自动调用 `reset_eye_movement()` / `reset_head_pose()`,清空平滑缓冲区与历史状态,避免上一次会话污染当前结果。

### 信号与数据流

```
DetectionThread
   │
   ├─ frame_ready(before, after)      → UI: 更新双画面
   ├─ status_update(head, eye, mobile) → UI: 更新状态灯
   ├─ cheat_detected(type, detail, img)→ UI: 计数++ / 写Excel / 保存截图 / 日志
   ├─ log_message(level, text)         → UI: 彩色日志
   └─ finished_signal                  → UI: 恢复按钮状态
```

## 十、常见问题

### Q1: 启动报错 `FileNotFoundError: 未找到 shape_predictor_68_face_landmarks.dat`

模型文件未放在可搜索路径,参考 [安装 - 下载模型文件](#3-下载模型文件)。

### Q2: dlib 安装失败

改用预编译版本:

```bash
pip install dlib-bin
```

### Q3: 摄像头打不开

- 检查是否被其他程序占用 (Zoom、腾讯会议等)
- Windows 下 `VideoCapture(0)` 失败可尝试 `VideoCapture(0, cv2.CAP_DSHOW)`
- macOS 需在"系统设置 → 隐私 → 摄像头"授权

### Q4: 视频播放速度太慢

调大"视频跳帧"。默认 2 帧 = 3x 加速,改为 5 即为 6x。

### Q5: 摄像头模式一直显示头部违规

很可能是校准不准:

- 校准阶段(前 1 秒)必须正对摄像头
- 可降低阈值敏感度:把头部阈值从 3.0 改为 5.0 秒

### Q6: 多人违规总在触发

- 检查是否有其他人在背景中入镜
- 若误检率高,可将"多人阈值"从 1.0 调高到 3.0

### Q7: 检测过程中能修改阈值吗?

可以调整 SpinBox,但新值只在**下次开始检测**时生效 (检测线程启动时读取参数)。想要热更新需要额外改代码。

### Q8: 没有 openpyxl 能用吗?

能用,但 "保存结果" 功能不可用。其他功能正常。装上即可:

```bash
pip install openpyxl
```

### Q9: 闭眼会不会一直触发警报?

不会。闭眼 (Eyes Closed) 被明确排除在瞳孔违规之外,只显示橙色状态提示,不计数也不写日志警报。

## 十一、扩展建议

- **录制完整检测视频** — 在 `DetectionThread` 里加 `cv2.VideoWriter` 把 `processed` 帧写出
- **多摄像头** — 把 `_pick_camera` 改成列出所有摄像头索引供选择
- **热更新阈值** — 用信号槽把 SpinBox 的值实时传给线程,线程中读取 `self.xxx_threshold`
- **网络推送** — 违规时通过 WebSocket / HTTP 推送到管理端
- **人脸身份校验** — 配合 face_recognition 库确认考生身份,防止替考
- **多路并发监考** — 一台服务器同时处理多个考生画面流,每路独立检测线程
